# SPR-0004B.3 Phase 2 — Validation

## Static checks performed while building the package

```bash
node --check dashboard/js/auth/pkce.js
node --check dashboard/js/auth/token-store.js
node --check dashboard/js/auth/auth-manager.js
node --check dashboard/js/providers/aws-backend-provider.js
node --check dashboard/js/app.js
```

Expected: all commands exit with code `0` and no output.

## Browser prerequisites

Use HTTPS or localhost. PKCE depends on the Web Crypto API, which browsers expose only in a secure context.

## Test A — Unauthenticated startup

1. Clear site data for the dashboard origin.
2. Open the dashboard.

Expected:

- `Anmelden` is visible.
- The cloud status says `Anmeldung erforderlich`.
- No request is made to `/api/vehicles` before authentication.
- No automatic redirect occurs.

## Test B — Authorization redirect

1. Select `Anmelden`.
2. Inspect the Cognito authorize URL.

Expected parameters:

- `response_type=code`
- configured `client_id`
- exact `redirect_uri`
- `code_challenge_method=S256`
- non-empty `code_challenge`, `state` and `nonce`
- configured scopes

The verifier remains in `sessionStorage` and is not placed in the URL.

## Test C — Successful callback

1. Complete Cognito sign-in.
2. Allow Cognito to redirect to the dashboard.

Expected:

- callback `state` matches the stored transaction;
- authorization code is exchanged at the token endpoint;
- access token is stored in `sessionStorage`;
- callback parameters are removed from the address bar;
- `Angemeldet` and `Abmelden` are shown;
- `/api/vehicles` is called with `Authorization: Bearer <access-token>`;
- the API returns HTTP `200` when Cognito and API Gateway are configured consistently.

## Test D — Reload

Reload in the same browser tab.

Expected:

- the non-expired session is restored;
- no Hosted UI redirect occurs;
- authenticated API calls continue.

Open the dashboard in a new browser session after closing the tab/window.

Expected: sign-in is required again because this sprint deliberately uses `sessionStorage`.

## Test E — Callback protection

Repeat a callback with a modified `state`, or clear session storage before the callback.

Expected:

- authentication is rejected;
- tokens are cleared;
- callback parameters are removed;
- no API provider starts.

## Test F — Token expiry

Set the stored `expiresAt` to a past timestamp and reload.

Expected:

- the session is removed;
- `Anmelden` is shown;
- the provider does not start.

## Test G — API 401

Cause the API to return HTTP `401` with a nominally non-expired browser token.

Expected:

- provider reports `AWS API HTTP 401`;
- UI states `Sitzung abgelaufen – erneut anmelden`;
- no token is printed to logs.

Phase 2 does not automatically redirect or refresh after a 401.

## Test H — Logout

Select `Abmelden`.

Expected:

- local session and pending transaction are cleared;
- browser redirects to the Cognito logout endpoint;
- Cognito redirects to the configured dashboard URL;
- the dashboard starts unauthenticated.

## Regression — Legacy MQTT

Set `dataSource.type` to `legacy-mqtt`.

Expected:

- authentication controls stay hidden;
- legacy MQTT behavior is unchanged;
- Cognito configuration is not required.
