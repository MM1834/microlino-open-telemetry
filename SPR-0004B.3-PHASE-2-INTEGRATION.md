# SPR-0004B.3 Phase 2 — Integration

## Preconditions

1. Work on `develop` or a feature branch, never directly on `main`.
2. The repository must contain the committed Phase 1 state used to create this package.
3. A Cognito User Pool app client must exist without a client secret.
4. The app client must allow Authorization Code grant and the scopes configured in `dashboard/config.js`.
5. Cognito callback and sign-out URLs must exactly match `auth.redirectUri`.
6. The token endpoint must be reachable from the dashboard origin. Cognito Hosted UI supports this browser PKCE exchange; a custom proxy is not part of this sprint.

## Apply the package

From the repository root:

```bash
unzip SPR-0004B.3-PHASE-2.zip -d /tmp/SPR-0004B.3-PHASE-2
rsync -av --exclude='SPR-0004B.3-PHASE-2-*' \
  /tmp/SPR-0004B.3-PHASE-2/ ./
```

Alternatively, review and copy each file listed in the manifest.

## Configure the dashboard

Copy the example only when no local `dashboard/config.js` exists:

```bash
cp dashboard/config.example.js dashboard/config.js
```

Set at least:

```js
auth: {
  region: "eu-north-1",
  userPoolId: "eu-north-1_...",
  clientId: "...",
  issuer: "https://cognito-idp.eu-north-1.amazonaws.com/eu-north-1_...",
  authorizeEndpoint: "https://<domain>.auth.eu-north-1.amazoncognito.com/oauth2/authorize",
  tokenEndpoint: "https://<domain>.auth.eu-north-1.amazoncognito.com/oauth2/token",
  logoutEndpoint: "https://<domain>.auth.eu-north-1.amazoncognito.com/logout",
  redirectUri: "https://<dashboard-host>/",
  scopes: ["openid", "email", "profile"]
}
```

The redirect URI is string-sensitive. Scheme, host, path and trailing slash must match the Cognito app-client configuration.

## Deploy

Deploy the static dashboard by the repository's existing procedure. No CloudFormation changes are included in this package.
