# SPR-0004B.2.3 — JWT claims in Vehicle API Lambda

## Change

The inline Python Lambda now reads JWT claims from:

`requestContext.authorizer.jwt.claims`

It normalizes `sub`, `cognito:username`/`username`, and `email`. For authenticated requests it writes a minimal structured log entry containing the subject and booleans indicating whether username and email claims are present. It never logs the JWT, authorization header, or complete claims object.

The public `/health` route remains usable without an authorizer context. Route authorization and API response bodies are unchanged.

