# SPR-0004B.3 Phase 2 — Changelog

## Added

- Cognito Hosted UI login and logout controls.
- PKCE S256 generation using Web Crypto.
- State and nonce transaction handling.
- Authorization-code callback processing and token exchange.
- Session restoration and access-token expiry checks.
- Authentication-gated AWS provider startup.
- Structured provider notification for HTTP 401.

## Changed

- Auth tokens now default to `sessionStorage` rather than persistent `localStorage`.
- AWS API requests obtain the bearer token from the authentication facade.
- Sprint and architecture documentation now describe the operational Phase 2 flow.

## Not added

- Refresh-token rotation.
- Roles or user-to-vehicle permissions.
- Cognito Identity Pool.
- Cognito/API Gateway CloudFormation resources.
