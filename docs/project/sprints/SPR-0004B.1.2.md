# SPR-0004B.1.2 – Cognito dashboard app client

## Goal

Add the public browser app client to the already deployed Cognito user pool without changing AWS IoT, telemetry ingestion, vehicle-state storage, or the Vehicle API.

## Scope

- Add `AWS::Cognito::UserPoolClient`.
- Do not generate a client secret.
- Enable only OAuth authorization-code flow.
- Prepare authorization-code flow with PKCE for dashboard implementation.
- Configure local callback and logout URLs through parameters.
- Add the client ID as a CloudFormation output.
- Document developer, administrator, and end-user perspectives.

## Out of scope

- Cognito domain / managed login
- login UI implementation
- API Gateway JWT authorizer
- user and fleet administration UI
- vehicle authorization table

## Acceptance criteria

1. CloudFormation validation succeeds.
2. The change set adds only `DashboardUserPoolClient`.
3. Existing resources are not replaced.
4. Stack update completes with `UPDATE_COMPLETE`.
5. `GenerateSecret` is `false`.
6. The only enabled OAuth flow is `code`.
7. Scopes are `openid`, `email`, and `profile`.
8. Callback and logout URLs match the supplied parameters.
9. `CognitoUserPoolClientId` is present in stack outputs.

## Rollback

If creation fails, CloudFormation removes the app client and restores the prior stack state. Do not retry until the failing stack event has been reviewed.
