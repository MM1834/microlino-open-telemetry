# SPR-0004B.1 – Cognito infrastructure

## Goal

Create the managed user-authentication foundation without changing firmware, AWS IoT telemetry, the Vehicle API authorization behavior or the dashboard UI.

## Scope

- Cognito user pool managed by the existing CloudFormation foundation stack
- Public browser app client without a client secret
- Email-based sign-in and verification
- Secure password and token defaults
- Public self-registration disabled by default
- Deletion protection enabled by default
- Documentation split for developers, administrators and end users

## Out of scope

- API Gateway JWT authorizer
- Protected Vehicle API routes
- `UserVehicleAccess` DynamoDB table
- Dashboard login and logout
- Vehicle assignment and fleet-administrator UI
- MFA rollout and external identity providers

## Acceptance criteria

- CloudFormation template validates.
- Change set contains a user pool and one public dashboard client.
- Existing AWS IoT and telemetry resources remain unchanged.
- App client has `GenerateSecret: false`.
- Self-registration is disabled with default parameters.
- User-pool deletion protection is active with default parameters.
- Stack outputs expose user pool ID, client ID and issuer URL.
- No credentials or tokens are added to Git.
- A test user can be created by an AWS administrator.
- Documentation has separate developer, administrator and end-user entry points.

## Handoff to SPR-0004B.2

Use the stack outputs to create the API Gateway HTTP API JWT authorizer. Keep `/health` public; protect versioned vehicle routes and require access tokens rather than trusting client-provided identity data.
