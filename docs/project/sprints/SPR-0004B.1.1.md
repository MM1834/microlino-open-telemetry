# SPR-0004B.1.1 – Cognito User Pool

## Goal

Deploy only the Amazon Cognito user pool as the first independently testable authentication increment.

## Scope

- `DashboardUserPool` in the existing CloudFormation foundation stack
- Email address as sign-in identifier
- Automatic email verification
- Administrator-created users only
- Password policy with a minimum length of 12 characters
- Account recovery through verified email
- MFA explicitly disabled with the string value `"OFF"`
- User-pool deletion protection controlled by the existing parameter
- User pool ID, ARN and issuer URL as stack outputs

## Out of scope

- Cognito app client
- Hosted UI or Cognito domain
- API Gateway JWT authorizer
- Dashboard login
- User-to-vehicle authorization
- Fleet-administrator interface

## Incident note

The first combined B.1 change set rolled back because unquoted YAML `OFF` was interpreted as the Boolean value `false`. Cognito accepts only the strings `OFF`, `OPTIONAL` or `ON` for `MfaConfiguration`.

The template therefore uses:

```yaml
MfaConfiguration: "OFF"
```

## Acceptance criteria

- The template validates.
- The change set adds only `DashboardUserPool`.
- No existing resource is replaced or modified unexpectedly.
- The stack reaches `UPDATE_COMPLETE`.
- The user pool reports deletion protection according to the selected parameter.
- The outputs `CognitoUserPoolId`, `CognitoUserPoolArn` and `CognitoIssuerUrl` exist.
- No app client exists yet.
- No credentials, passwords or tokens are committed.

## Next increment

SPR-0004B.1.2 adds the public dashboard app client without a client secret after the user pool has been deployed and verified independently.
