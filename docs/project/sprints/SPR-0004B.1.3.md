# SPR-0004B.1.3 — Cognito domain

## Goal

Expose the managed Cognito login and OAuth endpoints through one isolated CloudFormation resource.

## Infrastructure delta

```text
Add | DashboardUserPoolDomain | AWS::Cognito::UserPoolDomain | Replacement: None
```

## Included

- `DashboardUserPoolDomain`
- deterministic, account- and region-qualified domain prefix
- stack outputs for the base, authorization, token, and logout endpoints
- developer, administrator, and end-user documentation

## Excluded

- API Gateway JWT authorizer
- dashboard PKCE implementation
- user provisioning
- custom Cognito domain
- production callback URLs

## Acceptance criteria

1. `validate-template` succeeds.
2. The change set contains only `DashboardUserPoolDomain` as an addition.
3. The stack reaches `UPDATE_COMPLETE`.
4. The new domain resource reaches `CREATE_COMPLETE`.
5. `describe-user-pool-domain` returns the deployed user pool.
6. Existing user pool and app client remain unchanged.
