# SPR-0004B.3 Phase 2 — Release Notes

This sprint turns the Phase 1 authentication contracts into an operational Cognito PKCE login for the static dashboard.

After configuration, an unauthenticated user explicitly selects **Anmelden**, completes sign-in in the Cognito Hosted UI, returns with an authorization code, and receives a session-scoped access token. The AWS backend provider starts only after a valid session exists and adds the bearer token to Vehicle API requests.

The package contains dashboard code and documentation only. Cognito resources, app-client settings and the API Gateway JWT authorizer remain deployment prerequisites.
